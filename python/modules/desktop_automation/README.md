# 桌面自动化精准点击模块

> 如何在 2-3 次内获取准确坐标并执行点击操作

## 问题背景

在使用 AI 进行桌面自动化时，常见的坐标定位方法存在以下问题：

| 方法 | 问题 |
|------|------|
| 视觉AI估算坐标 | 偏差大，经常偏离目标 50-100 像素 |
| 全屏截图分析 | 无法精确读取像素坐标 |
| 窗口偏移计算错误 | 相对坐标转绝对坐标时出错 |
| 缺少验证机制 | 点击后才发现点错了 |

## 解决方案

**核心思路：截图 + 刻度网格 + 鼠标标注 + AI 验证**

```
第一步：截图 + 添加刻度网格
├─ 截取目标区域（局部，不要全屏）
├─ 添加坐标刻度线（每 25px 一条细线，每 50px 标注数字）
└─ AI 根据刻度精确读取坐标

第二步：移动鼠标 + 验证
├─ 移动鼠标到目标坐标
├─ 截图并在鼠标位置画十字准星
└─ AI 确认鼠标是否对准目标

第三步：点击 + 保存预设坐标
├─ 验证通过后执行点击
└─ 保存坐标到预设，下次直接使用
```

## 使用方法

### 方法 1：使用预设坐标（一次完成）

```python
from smart_click import SmartClick

# 初始化
clicker = SmartClick(window_title="抖音")

# 快速点击预设按钮
clicker.quick_click('精选')
clicker.quick_click('推荐')
```

### 方法 2：定位新按钮（2-3 次完成）

```python
from smart_click import SmartClick

clicker = SmartClick(window_title="抖音")

# 定位新按钮
result = clicker.locate_and_click('新按钮名称')
# 返回: {'success': True, 'coords': (88, 100), 'verified': True}
```

## 最佳实践

### 1. 截图区域要精确
- ❌ 全屏截图 → AI 无法精确读取坐标
- ✅ 截取目标按钮所在的小区域（如 250x350）

### 2. 刻度网格是关键
- 没有 AI 会估算，偏差 50-100 像素
- 有了刻度，AI 可以精确读取，偏差 < 5 像素

### 3. 验证机制不能省
- 永远先移动鼠标 → 截图验证 → 确认后点击

### 4. 预设坐标提高效率
- 第一次定位：2-3 次迭代
- 保存预设后：1 次完成，毫秒级响应

## 总结

| 场景 | 方法 | 次数 |
|------|------|------|
| 首次定位新按钮 | 截图+刻度 → AI分析 → 验证 → 点击 → 保存 | 2-3次 |
| 使用预设坐标 | 直接调用 quick_click | 1次 |

**核心要点**：
1. 截图要加刻度网格
2. 移动后要验证
3. 成功后要保存预设
